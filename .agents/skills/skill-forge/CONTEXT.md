# skill-forge

A self-improving meta-agent that discovers, devours, and creates AI agent skills.

## Language

**Skill**:
A SKILL.md file that gives an AI agent specialized instructions for a specific task. Portable across Cursor, Claude Code, Codex, Gemini, Windsurf.
_Avoid_: prompt, rule, instruction, template

**Devouring**:
The process of reading a skill deeply — extracting not just what it does, but WHY the creator designed it that way, what philosophy drives it, what it lacks, and what patterns transfer to other domains.
_Avoid_: reading, scanning, analyzing (too shallow — devouring implies absorption)

**Gap**:
A problem that developers experience repeatedly but no quality skill addresses. Not just "nothing exists" — also "what exists is lazy/mediocre/outdated."
_Avoid_: opportunity, idea, concept (too vague — a gap has PAIN behind it)

**Self-check**:
Mandatory quality gate run after every action. Root-causes own weaknesses and fixes them in the same session. Not optional. Not "a note for next time."
_Avoid_: review, assessment, retrospective (these happen after the fact — self-check is IMMEDIATE)

**Anti-laziness**:
The active search for what's MISSING when nothing seems wrong. "If it lacks something — find it. If nothing seems missing — find something to reveal what's missing."
_Avoid_: improvement, optimization (too passive — anti-laziness is AGGRESSIVE)

**Reputation**:
Stars, installs, forks, issues opened. The compound signal that a skill delivers genuine value. Every published skill must earn reputation or it's withdrawn.
_Avoid_: metrics, KPIs (corporate speak — reputation is personal)

## Relationships

- A **Devouring** produces learnings that update the knowledge base
- A **Gap** is identified by **Anti-laziness** scanning
- A **Gap** becomes a Skill when filled with quality (passes **Self-check**)
- A Skill earns **Reputation** or gets improved until it does
- **Self-check** triggers after EVERY action (non-negotiable)

## Flagged Ambiguities

- "skill" was previously used for both "SKILL.md file" and "capability" — resolved: Skill = the file/repo, capability = what it teaches
- "learning" was previously passive (store a note) — resolved: learning must change BEHAVIOR, not just exist in a file
